Project Management Analysis of an Agile Eye Solar Tracker using POM-QM
Project Overview

This project demonstrates the application of project management and operations research techniques to the development of an Agile Eye Solar Tracker. Using POM-QM for Windows, the project analyzes scheduling, risk, cost optimization, and production planning through CPM, PERT, project crashing, and assembly line balancing methods.

Objectives
Develop a project schedule for a solar tracker system.
Identify the project's critical path.
Estimate project duration using PERT.
Analyze project crashing to reduce completion time.
Optimize production using assembly line balancing techniques.
Problem Statement

Engineering projects often face delays, uncertain activity durations, budget constraints, and production planning challenges. This project applies quantitative project management techniques to improve scheduling accuracy, optimize resource utilization, and support efficient production planning.

Methodology
Defined project activities and precedence relationships.
Developed the Critical Path Method (CPM) network.
Estimated activity durations using PERT.
Performed project crashing analysis.
Balanced the assembly line using:
Longest Operation Time Method
Ranked Positional Weight Method
Shortest Operation Time Method
Interpreted project performance using POM-QM software.
